# Security Policy

Quick Boot 是一个后端脚手架项目，公开仓库中不应包含真实数据库密码、访问令牌、API Key、私钥或第三方平台密钥。

## 报告安全问题

如果你发现安全漏洞或疑似密钥泄露，请不要在公开 Issue 中披露细节。请通过 GitHub 私信或仓库维护者公开资料中的联系方式联系维护者，并提供：

- 受影响的模块或文件路径。
- 漏洞或泄露的影响范围。
- 复现步骤或最小复现信息。
- 你建议的修复方向。

维护者会优先处理密钥泄露、认证绕过、权限控制、数据泄露、任意代码执行、SQL 注入和配置默认值风险。

## 配置安全约定

- `application-local.yaml` 只用于本地开发，已经加入 `.gitignore`。
- 公开配置文件使用环境变量占位符，不提交真实密钥。
- 示例账号密码只能用于本地演示，生产环境必须通过环境变量或私有配置覆盖。
- 如果曾经提交过真实密钥，请立即轮换密钥，并在需要时清理 Git 历史。

## 自动化安全检查

仓库通过 GitHub Actions 执行基础安全检查：

- Pull Request、`main` 分支推送和每周定时任务会运行 Gitleaks，扫描提交历史和配置文件中的疑似密钥。
- Pull Request 会运行 Dependency Review，检查依赖变更中是否引入高危及以上漏洞，并在 PR 中输出依赖审查摘要。

如果检查报告误报，请在 PR 中说明原因，并优先通过环境变量、示例占位符或测试专用假值替换敏感字段。不要通过提交真实密钥来绕过检查。
